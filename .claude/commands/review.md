# /review — 記事のトーン・構成をレビューする

## 目的
指定された記事ファイルを読み込み、ブランドエディター視点でレビューする。

## 実行手順

1. 対象ファイルのパスを受け取る（未指定なら直近の下書きファイルを使う）

2. 以下の観点でレビューを行う：

### ✍️ 文体・トーン
- 「この記事を書いたのは誰か」が伝わるか
- 自分の言葉で書かれているか（AIっぽい表現がないか）
- `writing.md` の禁止表現が含まれていないか
- 読んでいて息が詰まらないテンポか

### 🏗️ 構成
- 導入 → 本題 → 締め の流れが自然か
- 締めに余韻や問いかけがあるか
- 見出しの使いすぎがないか

### 📏 分量
- カテゴリごとの目安文字数（`writing.md` 参照）に対して長すぎ・短すぎないか

3. レビュー結果を以下の形式で返す：
   ```
   ## レビュー結果：[記事タイトル]
   
   ### 良い点
   - （具体的に）
   
   ### 改善提案
   - （やさしく、理由とともに）
   
   ### 一言コメント
   （全体的な印象を一文で）
   ```

4. ユーザーが求めた場合のみ、具体的な文章の書き直し案を提示する

## 禁止事項
- レビューを求められていないのに記事を書き直さない
- 「よくできています」だけで終わらない（具体性を持たせる）

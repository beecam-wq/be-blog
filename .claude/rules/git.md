# Git コミット規約

## コミットメッセージ形式
```
<type>: <要約（日本語OK、50字以内）>
```

## type 一覧
| type | 用途 |
|---|---|
| `post` | 記事の新規追加 |
| `draft` | 下書きの追加・更新 |
| `fix` | 誤字・リンク修正など |
| `style` | デザイン・CSS調整 |
| `feat` | 機能追加（タグ一覧、検索など） |
| `chore` | 設定・依存関係の更新 |

## 例
```
post: 映画「ある視点」レビューを公開
draft: ディーター・ラムスの記事を追加
fix: トップページの画像パスを修正
style: モバイルでのフォントサイズ調整
```

## ルール
- `content/posts/` への変更は必ず `post:` または `draft:` を使う
- コミットは **1記事1コミット** を原則とする
- `git push --force` は禁止（`--force-with-lease` は可）
- `main` ブランチへの直接コミットは避け、`draft/` ブランチで作業する
